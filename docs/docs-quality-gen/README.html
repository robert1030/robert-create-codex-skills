docs-quality-gen 觸發與應用說明

docs-quality-gen 是 robert 個人的文件品質 gate。除了明確輸入 $docs-quality-gen，實際對話中只要出現與文件產出、同步、review、HTML 基礎美化或交付檢查相關的需求，也可能適合使用這個 skill。

主要觸發依據是 SKILL.md frontmatter 的 description。Skill 本文中的 workflow 與 hard rules 是觸發後才用來決定怎麼執行。

動作詞

這些動作詞通常代表使用者要處理文件品質、文件同步或文件交付。

分類 常見詞彙
產出與更新 產生文件、建立文件、更新文件、修改文件、整理文件、補文件
同步與一致性 同步、對齊、檢查前後一致、格式一致、流程一致、內容一致
Review 與交付 review、檢查文件品質、最終交付檢查、交付前檢查、品質 gate
HTML 文件處理 HTML 基礎美化、HTM 美化、表格分類、標題整理、條列式整理、色調美化
可讀性與可攜性 七年級可讀性、國一可讀性、本機環境提醒、Windows/Linux 指令對照

文件類型

這個 skill 主要處理 robert 個人的 SPEC、runbook、README，以及它們的 Markdown/HTML 版本。

  • SPEC.md
  • runbook.md、runbook.htm、runbook.html
  • README.md、README.htm、README.html
  • Markdown 與 HTML 成對文件
  • 需要交接、操作、review 或最終交付檢查的文件

對話例子

下列表達方式通常足以讓我判斷應該使用 docs-quality-gen。

使用者說法 代表的工作
幫我 review 這份 runbook.md 是否前後一致。 文件 review、流程一致性、內容一致性檢查。
同步 runbook.md 和 runbook.htm。 Markdown/HTML 同步。
把 README.html 做基礎美化，但不要改意思。 HTML 基礎美化與內容一致性檢查。
檢查 SPEC.md 有沒有把本機路徑寫成通用規格。 SPEC 規格邊界與本機環境可攜性檢查。
這份文件要補 Windows PowerShell 和 Ubuntu Bash 指令。 Windows/Linux 指令對照。
交付前檢查這三份文件。 最終交付 review。
檢查這份 runbook 國一生是否能看懂主線。 七年級可讀性檢查。

此 skill 的應用

docs-quality-gen 適合用在 robert 個人文件的產出與品質檢查，不是公開通用文件規格，也不是泛用寫作助手。

應用面向 實際處理內容
SPEC 文件 確認規格契約、範圍、非範圍、驗收條件與可驗證性。
Runbook 文件 確認每一步的目的、動作、預期輸出、驗證方式與流程順序。
README 文件 確認文件用途、主要入口、使用方式與交付前說明清楚。
Markdown/HTML 同步 確認成對文件的段落、警告、路徑、檔名、清單、表格與指令意思一致。
HTML/HTM 基礎美化 整理標題、條列式、分類表格、code block、提示區與基礎色調，並保持內容意思不變。
Windows/Linux 指令對照 在文件中分開提供 Windows PowerShell 與 Ubuntu Bash/sh 指令，並標示 WSL 路徑只是 WSL 範例。
可讀性檢查 讓交接或操作文件先說白話主線，再保留必要技術細節。
最終交付 review 確認格式、流程、內容、可讀性、同步狀態與「做了什麼、驗證了什麼、沒做什麼」。

不適用範圍

  • 不處理 Word .doc 或 .docx 文件。
  • 不加入 iTest 或其他專案專屬知識，除非該專案另有專屬 skill 或明確要求。
  • 不做完整 HTML 視覺設計、品牌設計、互動式 UI 或行銷頁。
  • 不主動打包 skill、安裝 skill、同步到 .codex\skills 或產生 zip，除非使用者明確要求。
