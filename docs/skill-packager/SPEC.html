skill-packager 規格書

版本：1.0.0　|　日期：2026-05-17　|　範圍：個人使用的 Codex skill 打包工具

目標

skill-packager 的目標是提供一個標準 Codex skill，用來驗證並打包其他 Codex skill。打包結果必須能支援本機個人安裝、$skill-installer 從 GitHub folder path 安裝，以及 zip 移植安裝。

非目標

  • 不做 Codex plugin。
  • 不做 marketplace 發佈結構。
  • 不修改 Codex 內建 .system/skill-creator。
  • 不把安裝教學放進 skill 本體。

目前產物

路徑 角色 是否進入 zip
skills/skill-packager/SKILL.md skill 定義與使用說明 是
skills/skill-packager/agents/openai.yaml Codex UI metadata 是
skills/skill-packager/scripts/package_skill.py 驗證與打包腳本 是
dist/skill-packager.zip 可移植安裝包 否，這是輸出成果
docs/skill-packager/README.html 白話使用與安裝說明 否
docs/skill-packager/SPEC.html 本規格書 否

標準 skill 結構

skills/
└── skill-packager/
    ├── SKILL.md
    ├── agents/
    │   └── openai.yaml
    └── scripts/
        └── package_skill.py

zip 輸出格式

輸出的 zip 必須保留頂層 skill 目錄。以 skill-packager 自身為例：

skill-packager.zip
└── skill-packager/
    ├── SKILL.md
    ├── agents/
    │   └── openai.yaml
    └── scripts/
        └── package_skill.py

對一般 skill，若存在 references/ 或 assets/，打包器也會包含它們。

安裝方式支援矩陣

安裝方式 輸入 預期結果 狀態
本機個人安裝 skills/skill-packager/ $CODEX_HOME/skills/skill-packager/SKILL.md 或 ~/.codex/skills/skill-packager/SKILL.md 支援
$skill-installer GitHub 安裝 GitHub repo 中的 skills/skill-packager/ folder path 安裝到 Codex user skills 目錄 支援
zip 移植安裝 dist/skill-packager.zip 解壓後得到 <skills-dir>/skill-packager/SKILL.md 支援

驗證規則

package_skill.py 在產生 zip 前會執行下列驗證：

  • 來源路徑必須存在，且必須是資料夾。
  • 來源資料夾內必須有 SKILL.md。
  • SKILL.md 必須有 YAML frontmatter。
  • frontmatter 必須能被 PyYAML 解析為 dictionary。
  • 允許的 frontmatter keys：name、description、license、allowed-tools、metadata。
  • name 必須是非空字串，只能使用小寫英文字母、數字和 hyphen。
  • name 不能以 hyphen 開頭或結尾，也不能包含連續 hyphen。
  • name 長度上限為 64 字元。
  • skill 資料夾名稱必須等於 frontmatter 的 name。
  • description 必須是非空字串，不能包含 angle brackets，長度上限為 1024 字元。

排除規則

打包時會排除下列內容：

類型 排除項目
目錄 .git、.hg、.svn、__pycache__
檔名 .DS_Store
副檔名 .pyc、.pyo、.zip

使用介面

python3 skills/skill-packager/scripts/package_skill.py <path/to/skill-folder> [output-directory]
python3 skills/skill-packager/scripts/package_skill.py <path/to/skill-folder> --output-dir dist

本機安裝指令

Linux、WSL、macOS

mkdir -p "${CODEX_HOME:-$HOME/.codex}/skills"
cp -R skills/skill-packager "${CODEX_HOME:-$HOME/.codex}/skills/skill-packager"

Windows PowerShell

$skills = if ($env:CODEX_HOME) {
  Join-Path $env:CODEX_HOME "skills"
} else {
  Join-Path $HOME ".codex\skills"
}

New-Item -ItemType Directory -Force $skills
Copy-Item -Recurse -Force .\skills\skill-packager (Join-Path $skills "skill-packager")

$skill-installer 安裝指令

將 skills/skill-packager/ 放到 GitHub repo 後，使用：

$skill-installer install https://github.com/<owner>/<repo>/tree/main/skills/skill-packager

zip 安裝指令

產生 zip

python3 skills/skill-packager/scripts/package_skill.py skills/skill-packager dist

Linux、WSL、macOS 解壓

mkdir -p "${CODEX_HOME:-$HOME/.codex}/skills"
python3 -m zipfile -e dist/skill-packager.zip "${CODEX_HOME:-$HOME/.codex}/skills"

Windows PowerShell 解壓

$skills = if ($env:CODEX_HOME) {
  Join-Path $env:CODEX_HOME "skills"
} else {
  Join-Path $HOME ".codex\skills"
}

New-Item -ItemType Directory -Force $skills
Expand-Archive .\dist\skill-packager.zip -DestinationPath $skills -Force

後續維護原則

  • 維持 skill-packager 是 skill，不升級成 plugin。
  • 文件與安裝教學維持在 dist/，不要放進 skill 本體。
  • 如果 Codex skill frontmatter 規則變更，優先同步 package_skill.py 的驗證規則。
  • 如果要公開給別人用，優先把 skills/skill-packager/ 放在 GitHub repo，讓 $skill-installer 可直接安裝。
